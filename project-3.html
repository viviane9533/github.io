NBA Dataset

In my computer science class, my group and I created a web application provides a user-friendly interface to navigate a complete NBA roster along with providing statistical insights and graphics. This is a Web-based data analytics application built on a Javascript/React/Python stack.

Code Link

Company - UC Riverisde Computer Science Class

Role - UI/UX Designer and Frontend Developer

Focus - Interface Design

In my Introduction to Software Engineering class we created a project. This is a team project (5 students per team). The goal is to build a data analytics app mostly from scratch (i.e., without using any database or analytics library). The app provides predefined data analytics for gaining insights from the data. Initially, the app can be populated with some public dataset in CSV format.

Task

We need a data analytics application to present the NBA Dataset (below). Also use data analytics to show visual representation of the data. The in-memory data store should follow a layered client-server architecture as shown below

Features

We need had to implement many features as a requirement for our project. Below are all the fully functioning features we successfully implemented!

Design

I was responsible in creating each Interface for the project. We wanted a user-friendly and appealing interface which is easy to navigate. I start off creating wireframes on Miro.


The Homepage is where the user can access all the features of our application. We have a sidebar which accesses the three analytic graphs. Starting at the top we have a toggle button that switches between the Home table and the MVP table. Then we have a search bar where you can search up any player you want. Below we have two drop down filters that will filter the data in the table. The first filter will filter the Year and the second filters based on Teams. Once you’ve inputted those parameters you click the search button and it the page will update with a table that it filtered based on your selections.


These are the pop up for the add player and edit player features. Once you click on those buttons the pop up will appear. The add a player pop up requires users to input the necessary information. The edit player pop up is basically the same but I wanted the player’s original data to be prefilled on the form so you can see what is being edited.


Here we have the MVP table that you can access using the toggle button at the top. It will show the table of MVP’s of Basketball since 2003. We also implemented a pagination at the bottom of the page that divides the data set into pages that you can access through the pagination bar below. We can see we have a lot of data that fills 290 pages! It will updated given the searches or edits or additions you make.


This is the UI of our Analysis Data. The first is the analysis of NBA Players’ Career Length. It shows up in a pop up. A cool feature on all our graphs and you can see on this graph is when you hover over a part of the graph you can see the data displayed. So my cursor was on the 3rd bar in the graph and it shows the data associated with that part of the graph.


These are the two other Analysis Data graphs. The pie chart is of the NBA Players Career Length Scale and the other is shows the top 10 NBA Players. In our NBA web application, we have sideBar component, searchBar component, pie chart component, bar chart component and two table components. We use these components to split the user interfaces into independent and reusable parts.

Challenges

Our group had a hard time implementing the data flows. After multiple TA office hours here's how we solved it. Data flows to the frontend via get requests. They call a specific backend API which gathers the requested data from the cache to transfer. Data flows to the backend via Posts, in which the frontend passes comma delimited strings and operation type to the backend to parse out.

Accomplishments

This program has many awesome features that were challenging but rewarding to implement. We have fully functioning add, edit, and delete features. Also the analytic graphs clearly represents our data in a concise way. I'm proud of how it looks. We received great feedback from our professors of how the UI of our project looks good and it easy to follow.

What's Next For NBA Dataset

We hope to add more features such as sharing and exporting the data set tables we created. We also want to be able to learn more about the players in the data set by being able to navigate to their own personal social media account.

Overview

My Introduction to Software Engineering class final project consisted of my team and I creating a data analytics application to present the NBA Dataset. Along with that, we implemented features such as Add, Edit, Delete, and many more.
